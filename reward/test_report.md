## 步骤得分设计验证

### 纯动作步骤
- **CONCENTRATE** 等无信号步骤：动作对了就得满分（1分）
- 验证通过：R_step = 1.0

### 有信号步骤
- 分子+用量+条件 = 3分
- 只有条件 = 1分
- 只有分子+用量 = 2分

### 归一化计算
```
R_step = 实际得分总和 / 理论最大分值总和
```

## 时间/温度解析支持

### 时间格式
- ✓ 单个值: "15 h", "30 min", "2 day"
- ✓ 范围: "2-3 h", "3-5 day"
- ✓ below/above: "below 3 min", "above 2 h"
- ✓ 自然语言: "overnight"

### 温度格式
- ✓ 单个值: "25°C", "25 C", "100C"
- ✓ 范围: "20-25°C", "3-5 C"
- ✓ below/above: "below 40°C", "above 0 C"
- ✓ 自然语言: "room temperature", "rt", "reflux", "ice bath"

## 奖励函数行为分析

### 预期行为
1. **完美匹配**: ~0.99分 ✓
2. **格式错误**: <0.5分 ✓
3. **序列错误**: 根据偏离程度 0.6-0.9分 ✓
4. **内容错误**: 根据错误严重程度 0.2-0.8分 ✓
5. **完全胡言乱语**: <0.1分 ✓

### 关键特性
1. **双向惩罚**: 漏步和啰嗦都会降低LCS得分
2. **信号自适应**: 理论最大值基于GT实际信号数
3. **纯动作满分**: 无信号步骤动作对了就得满分
4. **阈值判断**: 分子Jaccard>=0.5，用量相似度>=0.8
